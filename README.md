# oppProject
Целью данной курсовой работы является разработка программного приложения «MediaLibrary», предназначенного для хранения и систематизации фильмов и сериалов, предоставляющего механизм поиска по названию и отображение списка элементов в удобной форме. Приложение  позволяет  добавлять новые записи, просматривать  сохраненные и искать в каталоге.

Возможности приложения

Добавление фильмов и сериалов

Указание названия, года, жанра, режиссёра или количества сезонов

Удаление элементов по названию

Поиск по ключевому слову

Просмотр всей медиатеки

Автоматическая загрузка при старте

Сохранение данных вручную + автосохранение при закрытии

Красивый интерфейс на Swing

Поддержка всех принципов ООП

 Архитектура проекта
Проект разделён на 3 основных слоя:
1. Model (модели данных)
MediaItem — абстрактный класс (название, год, жанр)
Movie — фильм (режиссёр)
Series — сериал (количество сезонов)
2. Storage (хранилище)
MediaLibrary
хранение списка объектов
добавление/удаление
сериализация в файл
загрузка данных
3. UI (графический интерфейс)
MediaLibraryGUI
ввод данных
кнопки добавления, удаления, поиска
отображение результатов
кастомные градиентные кнопки
обработка событий

Принципы ООП, использованные в проекте
Инкапсуляция

Все поля моделей — private, доступ через геттеры.
Наследование

Movie и Series наследуют MediaItem.
Полиморфизм

Все элементы хранятся в List<MediaItem>,
метод getInfo() переопределён в каждом подклассе.
Абстракция

MediaItem — абстрактный родитель для всех медиа-объектов.
Полиморфизм интерфейсов

MediaLibrary implements Serializable.

Сохранение данных
Данные хранятся через механизм сериализации Java:
список сохраняется в файл catalog.dat;
при запуске программа загружает данные автоматически;
при закрытии приложения происходит автосохранение.

Запуск проекта
1. Скомпилировать проект
javac *.java
2. Запустить приложение
java MediaLibraryGUI
